# Bahria Assignment Tracker

Automation script that scrapes Bahria University LMS assignments and syncs new ones to Google Calendar with reminders. Runs on a 6-hour schedule via GitHub Actions and keeps a local SQLite database to avoid duplicate events.

## How it works
- Logs into the CMS/LMS, scrapes assignments, and skips anything already submitted or past due.
- Normalizes deadlines, then creates calendar events with four reminders (1440, 180, 60, 15 minutes).
- Stores fingerprints in `assignments.db` to prevent duplicate calendar entries.
- Scheduled workflow (`.github/workflows/sync.yml`) runs every 6 hours and can also be triggered manually.

## Local setup
1. **Requirements**: Python 3.11 (officially tested; newer 3.11+ versions are unverified), Google Calendar API enabled in your Google Cloud project.
2. **Install dependencies**:  
   ```bash
   pip install -r requirements.txt
   ```
3. **Google OAuth credentials**: Download your OAuth client `credentials.json` into the repository root.
4. **Generate `token.json`** (one-time):  
   ```bash
   python auth.py
   ```  
   Follow the browser prompt. The refreshed token will be reused by `sync.py`.
5. **Run the sync locally**:  
   ```bash
   BAHRIA_USER="<your_erp_id>" \
   BAHRIA_PASS="<your_password>" \
   python sync.py
   ```

## Deploy with GitHub Actions
The scheduled workflow already exists at `.github/workflows/sync.yml` (runs every 6 hours).

Add these repository secrets:
- `BAHRIA_USER` – your ERP ID  
- `BAHRIA_PASS` – your LMS password  
- `GOOGLE_CREDENTIALS_B64` – base64 of `credentials.json`  
- `GOOGLE_TOKEN_B64` – base64 of `token.json` (generated by `auth.py`)

You can generate the base64 values with:
```bash
python encode_secrets.py credentials.json
python encode_secrets.py token.json
```

## Repository files
- `sync.py` – main scraper and Google Calendar sync script
- `auth.py` – runs OAuth flow to create `token.json`
- `encode_secrets.py` – helper to base64-encode secrets for GitHub
- `requirements.txt` – Python dependencies
- `.github/workflows/sync.yml` – scheduled GitHub Actions workflow

## Troubleshooting
- Check `cron.log` (written to the repo root both locally and in GitHub Actions) for run output and `debug_*.html/txt` files (if present) for captured LMS pages.
- In GitHub Actions, `cron.log` lives in the workspace for that run but is not uploaded automatically—use the step logs for quick inspection or upload the file as an artifact if you need to retain it.
- If events stop appearing, regenerate `token.json` with `auth.py` and update `GOOGLE_TOKEN_B64`.
